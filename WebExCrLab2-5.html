Web Extra Credit Lab 2.5: CSS Redux

THIS IS FOR IF THEY USE THE UBUNTU DIRECTORY

This week, you will use CSS to style the appearance of the files on your server. If you go to your home directory right now, you will see a basic web page with a table listing the contents of the directory. This is the default page that is generated by Apache, the web server that is set up. In this lab, you will use CSS to make this page look a bit fancier while still displaying all the same information.

Make a file called htaccess.txt on your Desktop and copy and paste the following into it: 

Options +Indexes

IndexOptions +Charset=UTF-8 +FancyIndexing +IgnoreCase +FoldersFirst +XHTML +HTMLTable +SuppressRules +SuppressDescription +NameWidth=* +IconsAreLinks

The first line turns on Apache's automatic index generation, meaning it creates the default file view. The second line enables specific options for how the information should be displayed on the page. Now add the following line to your file but substitute in your username. This line tells Apache where to look for the CSS styling rules you will write later in this lab.

IndexStyleSheet "/~{username}/style.css"

These are all the options you need to set.

Now make a file called style.css and save it to your desktop. Write CSS rules to change the appearance of the default index page. Some tags to try could be styling the hyperlinks, or coloring the cells of the table.

Move both files into the public_html folder in your home directory with the scp command. 

scp Desktop/htaccess.txt {username}@52.27.148.155:public_html
scp Desktop/style.css {username}@52.27.148.155:public_html

You will be prompted for your password each time.

Even though the files are in the correct place, they still have to be configured a bit. When Apache looks for customized display options, it will look for a file named .htaccess (not htaccess.txt as this lab named it). Sometimes files that lead with a dot do not show up to the user and can be easy to lose, which is why these directions created it as a text document first. Now, you will need to change the file type with a remote login. Substitute your username into the following command:

ssh {username}@52.27.148.155

Navigate to the public_html folder:

cd public_html

Make a copy of the contents of htaccess.txt and rename it to be .htaccess

cp htaccess.txt .htaccess

In the future, if you try to view the contents of this folder, this file will not be visible, but it is still there. You can edit the htaccess.txt document and recopy it.

Make this file viewable on the web so it can run in your browser:

chmod 755 .htaccess

Every file that you add has to be made public with chmod 755 <filename> in order to be accessed via the browser. If you're interested, you can read more about file permissions here: https://www.maketecheasier.com/file-permissions-what-does-chmod-777-means/. Lastly, make the style.css file viewable: 

chmod 755 style.css

Now view your home directory with a browser and see your CSS modifications
