Extra Credit Lab 10: Enter the Picobot

In this extra credit lab, you will implement

  • HTML/HTML5 Canvas
  • Javascript
  • CSS
to create a web based Picobot app, as used in the first homework assignment.

The starter files are picobot.html and picobot.js

An Intro to HTML5 Canvas

Why get excited about HTML5 Canvas? Several super cool web apps use Canvas because it is great for games and animation- the most notable examples include iPhone app Sinuous, images which use 8-bit color-cycling a'la retro gaming, Bomomo, Litesaber Training, and the strangely hypnotic liquid characters.

The HTML5 Canvas is an element that you can insert into your HTML document. We would include our own tutorial here, but a good tutorial has already been done here at diveintoHTML5.

We strongly suggest that you try out everything in the first half of the tutorial.

Sample Interface

We only implemented the empty room: you should feel free to implement the other maps for even more extra credit!

HTML Details

You will be provided with picobot.html, which contains Picobot's basic functionality. Try spicing up the CSS, and if you decide to include teleport or other features, include the user interface for those.

A useful resource for creating banners and other images is Pixlr, on online image manipulation program.

Javascript Details

You will start with picobot.js, which includes all of the basic functions (with some blanks!) that you will need for implementing the first level of Picobot.

  • Write reset(), which should move the Picobot to a random location on the board, reset the state, set all squares on board to empty (except walls) and redraw the maze.

  • Write drawMaze(), which will use the same techniques as the HTML5 tutorial above. It will be very useful to loop over all of the columns and rows in the maze and deal with all possibilities accordingly. Here is also where we called countEmpty(), which counted all of the empty (unvisited) cells.

  • Write pause() and resume(). For these, you will need clearInterval and setInterval(move, time). These should each be one line functions.

  • Write loadrules(), which should take input from the PicoRules? text box and parse it. You should ignore comments that begin with #, and sort the starting state, surroundings, direction, and ending state into the rulesArray. Once all the rules are separated, it may be helpful to split rules on the space character.

  • Write getSurroundings(r,c), which takes in the row and column number of any square on your board and returns the environment in Picobot surroundings form, eg, Nxxx.

  • Write getDirect(), which should call getSurroundings(), then check the current surroundings against your rules array (and current state) to determine the correct direction for the Picobot to travel (and perhaps state change as well).

  • Write move(), which should get the Picobot's direction from getDirect(), update the location of Picobot (depending on the direction), set that cell in the maze to 'Picobot', and set the cell where Picobot was previously as visited.

  • Write countEmpty(), which loops through all of the cells in the maze and keeps a running tally of the empty ones. Needless to say, if there are no empty cells, then you have won the game!

  • Bonus! Write teleport(direction), which looks up the current bot location, resets the maze (so picobot's path won't carry through), then moves the Picobot. If you implement teleport, then you should also add buttons on your html page that call this function.

Of course, there are also many other features that you can implement! Run Pico-wild!
