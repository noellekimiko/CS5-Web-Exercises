Extra Credit Lab 5: File Upload and Analysis

Your goal in this assignment is to create a small web application that takes in an uploaded text file and analyzes it. For this week, you may assume that the file is, in fact, an ASCII text file.

File Statistics

To Do: Implement a web application that uses
  • HTML + CSS
  • CGI
and allows an end user to upload a file and receive statistics about that file.

Here is an example of what your interface might look like:


Note that the file has 4 lines because the last line is just a plain newline.

HTML Details
In this assignment we would like you to use HTML + CSS to construct the pages and make them look presentable. You will use CGI to allow your front page to call the python script you have written. This is also an opportunity to learn how to upload and save files using HTML and CGI.

Here is the HTML for the front page; it will look identical to the example. The second page will be HTML printed by your Python script. You should change the link tag to use your own favorite CSS style sheet. 

<html>
<head>
<link rel='stylesheet' type='text/css' href='mystyle.css' />
<title>File Upload and Analysis</title>
</head>
<body>
<h2>Please upload a file for analysis:</h2>
<form enctype="multipart/form-data" action="upload.py" method="post">
<p>File: <input type="file" name="file"></p>
<p><input type="submit" value="Upload"></p>
</form>
</body>
</html>
Note that the python script that this code calls in the the action attribute of the form is named upload.py.This is the easiest way to use CGI.

As always, make sure that your files have appropriate permissions to let the webserver call them. Scripts must be executable, html files and images must be readable.

Python Details
Write code inside of upload.py to upload the file and analyze its contents.
  • Don't forget to have
    • #!C:/Python27/python.exe for Windows users, or
    • #!/usr/local/bin/python for Mac users
    as the first line of your python file. This allows your python scripts to run in CGI.
  • Don't forget to have 
    import cgi
import sys
import os
import cgitb; cgitb.enable()
    after the first line, at the top of your python file.
    • The cgi module allows you to access the data from the form you will use to call the python script.
    • The sys module allows your python script to abort.
    • The os module helps handle the filenames.
    • The cgitb module will allow python error output to be displayed on your web browser if something goes wrong. It is very helpful for troubleshooting.
  • Don't forget that your script's output must start with 
    print "Content-Type: text/html; charset=ISO-8859-1\n\n"
print
Handling File Upload
Below is CGI code to save the uploaded file. If you need more information, a good reference for file upload can be found at http://webpython.codepoint.net/cgi_file_upload.

Code to save uploaded file: 

form = cgi.FieldStorage()

fileitem = form['file']

if fileitem.filename:
   
   fn = os.path.basename(fileitem.filename)
   open('files/' + fn, 'wb').write(fileitem.file.read())
   
else:
   sys.exit(0)
This will save the uploaded file into a new directory named files, which is located in the same directory as your upload.py file. The files directory must already exist otherwise this script will not work. Alternatively, you could just not put the uploaded files into a subdirectory if you would like by changing the open line to open(fn, 'wb').write(fileitem.file.read()).

Analysis: Part 1
Note: For the analysis functions, we would like you to do your debugging by first making sure your program works from the Python shell before implementing it in CGI.

Your python script will need to:

  • count all of the words in the file
  • count all of the lines in the file
  • count all of the characters in the file

Then, print out those three numbers at the top of your output page.

Be sure to try this out with one or two small files -- and check that the answers are correct!

Details such as counting blank characters (spaces, newlines) are up to you to decide. Any consistent approach is acceptable. For example, in the interface above, space characters and newline characters are counted. In fact, there is a blank line at the end of the file, which is why it reports four lines instead of three.

Analysis: Part 2
For the second part of this project, add a list of words from the file and their frequencies below the information from part 1.

For full credit, you should output the words in decreasing order of frequency, so that the most common appearing word is first, the second most common word is second, and so on. Most credit (80%) is available for a result that has all of the words in any order at all.
