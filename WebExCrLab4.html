Extra Credit Lab 4: JavaScript and The Caesar Cipher

In cryptography, a Caesar Cipher is one of the simplest and most well known encryption techniques. It is so named because it was used by Julius Caesar to communicate with his generals who were spread across the Roman empire. The basic idea of the caesar cipher is to shift every letter in a message by a predetermined number. Thus, a caesar cipher with a shift of 3 would transform A to D, B to E, etc... You can read more about the caesar cipher here: http://en.wikipedia.org/wiki/Caesar_cipher

To Do: Implement a web application that uses
  • HTML + CSS
  • JavaScript
and allows an end user to specify a message to encrypt and how far to shift it by, then shows the encrypted message.

Python Details

Note: You should have written encipher in Homework 3 Problem 2 but if you do not already have it the details are below.

encipher( S , n ) takes as input a string S and a non-negative integer n. This function returns a new string in which the letters in S have been rotated by n characters (note that if n is greater than 26 the shifting will wrap. The modulus operator, %, could be very useful here). For this problem, you should assume that upper-case letters are "rotated" to upper-case letters, lower-case letters are "rotated" to lower-case letters, and all non-alphabetic characters are left unchanged. For example, if we were to shift the letter 'y' by 3, we would get 'b' and if we were to shift the letter 'Y' by 3 we would get 'B'. (In python, you can use the test if 'a' <= c <= 'z': to determine if a character c is between 'a' and 'z' in the alphabet.)

You can write encipher any way you like, using functional programming, loops, or recursion. You may wish to write a helper function that "rotates" a single character by n spots, wrapping around the alphabet as appropriate. Then, you might use this helper function to encipher your string. It's up to you how you do this! (Remember that uppercase letters wrap around the alphabet to uppercase letters, and lowercase letters wrap always to lowercase letters.)

Remember that testing your python functions before you move on to the rest of the assignment is always a good idea. You might want to check independently that encipher is working and also that the final script prints out the HTML that you think it will.

As a reminder, the built-in functions ord and chr convert from single-character strings to integers and back. For example, ord('a') outputs 97 and chr(97) outputs 'a'. These functions will make this problem much easier!

Some encipher examples: 

>>> encipher('xyza', 1)
'yzab'

>>> encipher('Z A', 1)
'A B'

>>> encipher('*ab?', 1)
'*bc?'

>>> encipher('This is a string!', 1)
'Uijt jt b tusjoh!'

>>> encipher('Caesar cipher? I prefer Caesar salad.', 25)
'Bzdrzq bhogdq? H oqdedq Bzdrzq rzkzc.'

JavaScript

In the previous lab we handled form data by calling a python script which would create a new HTML page. However, if we simply want to change a small portion of a page it becomes infeasible to send information back to the server and change a detail and recreate the rest of the page. This is where JavaScript comes in handy. JavaScript allows for these changes to be made on the user's end. For this lab we will handle our form data using JavaScript and return the result to the page.

Create an HTML document named caesarcipher.html. Include the following script tags in this document. They can be placed in the head, body or you could use an external JavaScript file. For now I recommend putting them right before the end of the body. 

<script >
... some JavaScript code ...
</script>
These script tags will be where you will put all of your JavaScript code.

JavaScript Syntax

  • Line endings: regular lines (meaning a line of code that is not right before opening a block) have to have a semicolon at the end. Take care to add these in when converting your python code.

  • Functions 
    function encipher()
{
...
}
    Notice that we have removed the arguments from the encipher function. You should use id or name attributes when creating the input elements of your form. Then, you will get the information from these elements inside of encipher (more on this later).

  • If/Else 
    if (condition)
{
 code to be executed if condition is true
}
else
{
 code to be executed if condition is not true
}
  • For Loops 
    var i=0;
for (i=0;i<=5;i++)
{
  document.write("The number is " + i);
  document.write("<br />");
}
    Note: In the above example we had to declare the variable before using it for our loop by saying var i. All variables must be declare in this way, using var, before they can be used. This can also be done within the loop by saying, 
    for (var i=0; i<5; i++)
{
  document.write("The number is " + i);
  document.write("<br />");
}
  • Getting Elements
    When using JavaScript you can refer to specific elements of the HTML in order to make changes. For this to work you must first make sure your HTML tag for the element has an id. It is important to note that ids are unique and two elements on the page should not have the same id. For instance, take our paragraph with the id="result" from before. Then to access this element in JavaScript you can use the following line. 
    document.getElementById('result');
    You will then have the ability to modify parts of the element using methods like .innerHTML or .src. This will be useful in getting the information out of your form

  • Getting Form Information
    You can reference information that was submitted in the form using the name attributes. If you used the HMTL form provided, these two lines will be: 
    var toEncipher = document.forms["myForm"]["toEncipher"].value;
var shiftStr = document.forms["myForm"]["shiftVal"].value;
    This code will be useful inside of encipher(). It is important to note that these values are both strings by default. Make sure to convert the shift value to an integer! (Try the parseInt command.) Otherwise, the shift value it will be treated as a string and can cause unexpected results.

  • Events
    Another import part of JavaScript that you will need to know are events. Events allow you bind some action with a function. Some examples of events are, keystrokes, a mouse click, a mouse over or submitting a form. This last one will mostly likely be the one you want to use. For this you will want your form tag to look something like this: 
    <form name="myForm" method="get" action="" onsubmit="return encipher()">
    Where encipher() is the function you want to bind to the onsubmit event and is already written within your script tags. When writing encipher(), make sure that it has the line return false; at the very end; this will prevent the page from reloading once the script runs.

With this syntax in mind, convert your python encipher code to JavaScript. Then create the webpage interface with HTML, and call your function with the code directly above.
