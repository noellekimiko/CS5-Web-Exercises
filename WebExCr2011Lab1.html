Web Extra Credit Lab 1 (HTML)

Running a Simple CGI Web Server

Before making our webpage, we will set up a CGI server from the command-line. This server can be used to host your webpage locally so it is easier to view and update. In case you are not familiar with command-line prompts: http://mally.stanford.edu/~sr/computing/basic-unix.html

First create a directory (folder) on the desktop called web. Inside web, create another directory called cgi-bin. The web directory is where you will to save your webpage later in this lab. Now, inside of cgi-bin make a new python file named hello.py. Copy and paste the following content into this file, and then save it: 

#!/usr/local/bin/python

print 'Content-Type: text/html'
print
print '<html>'
print '<head><title>Hello from Python</title></head>'
print '<body>'
print '<h2>Hello from Python</h2>'
print '</body></html>'

This file will help you to see if your files are running on the server. In order to view it in a browser, there are certain permissions that have to be set. Open up the command-line prompt (or Terminal on a mac) and navigate to the cgi-web folder by running: 

cd Desktop/web/cgi-web

Then set the permissions to make this file world accessible with the chmod command: 

chmod 755 hello.py

Navigate back up to the Desktop/web folder: 

cd ..

Paste in the following to open up python. 

python -m CGIHTTPServer
You should then see the following message. 
C:\classes\python>python -m CGIHTTPServer
Serving HTTP on 0.0.0.0 port 8000 ...

The server is now running your page! To view it, open http://localhost:8000/cgi-bin/hello.py in your browser.


Troubleshooting...

Getting an Error Message when starting up your server? If you run into problems hosting your website, the problem might be in the first line of hello.py. Check where your version of Python is saved, and change the file path as necessary. If the command gives you a message like: 

'python' is not recognized as an internal or external command,
operable program or batch file.
or: 
Bad command or filename

...Then you probably need to set your computer to know where the Python interpreter is. Read this PythonPath page to learn how.



Otherwise, the following type of messages should appear in your command line window if things are going well: 

localhost - - [29/Jan/2008 20:14:17] "GET /cgi-bin/hello.py HTTP/1.1" 200 -
localhost - - [29/Jan/2008 20:14:17] command: C:\Python25\python.exe -u C:\class
es\python\cgi-bin\hello.py ""
localhost - - [29/Jan/2008 20:14:17] CGI script exited OK

If at any time you need to exit the server, type Control-C.

Creating an HTML document

While keeping your server open, create the html document for your webpage. To do this, open a new document in Sublime, and then save it as problem1.html in your web folder. Note that this is a different location from where the hello.py file is saved.

This new file has to be world accessible and so we use the chmod command from inside the web folder: 

cd ~/Desktop/web/
chmod 755 problem1.html

To view it, you will use the following address in your browser: http://localhost:8000/problem1.html. Right now, there is no content at that page, but you will add content in the upcoming steps.

Body and Headers in HTML

HTML uses tags to describe the information on a webpage. Note that opening and closing tags are required for every html page. Tags with a forward slash are used to close open tags- if you forget a closing tag, your page will not turn out the way you want it!

Header tags, such as h1, h2, and paragraph tags p, distinguish text on the page.

You can copy and paste this code into your problem1.html file to see how these tags work. 

 
<html>
<body>

<h1>My First Heading</h1>

<p>My first paragraph.</p>

</body>
</html>

There are also tags for basic text formatting: 

<p><b>This text is bold</b></p>
<p><strong>This text is strong</strong></p>
<p><big>This text is big</big></p>
<p><i>This text is italic</i></p>
<p><em>This text is emphasized</em></p>
<p><code>This is computer output</code></p>
<p>This is<sub> subscript</sub> and <sup>superscript</sup></p>

Feel free to try these out in your file! Make sure to put this code inside of the body section of the page.

Lists

There are two types of lists in HTML, ordered lists and unordered lists. Ordered lists number the items, while unordered lists use bullet points.

Unordered lists start with a ul tag, and ordered lists start with a ol tag.

Individual items in the list are denoted by the li tag, which stands for 'list item'.

Here is an example of an unordered list: 

<ul>
<li>aliens</li>
<li>spam</li>
</ul>

which will end up looking like:

  • aliens
  • spam

Here is an example of an ordered list: 

<ol>
<li>aliens</li>
<li>spam</li>
</ol>

which will be:

  1. aliens
  2. spam

Include a list of either type inside your problem1.html file.

Tables

Tables are denoted by the table tag. The tr tag means "table row" and within rows are any number of td (table data) tags that will be the columns. 

<table>
<tr>
<td>row 1, cell 1</td>
<td>row 1, cell 2</td>
</tr>
<tr>
<td>row 2, cell 1</td>
<td>row 2, cell 2</td>
</tr>
</table>
row 1, cell 1 row 1, cell 2
row 2, cell 1 row 2, cell 2

Add a table to your problem1.html file as well.

Links

The html code for creating a hyperlink is fairly straightforward: 

<a href="url">Link text</a>

The a tag stands for 'anchor' and the string specified with the href attribute determines the destination of the link.

To change the code above, replace the url and Link text with your desired values.

Images

The html tag for images is img. These tags are a special type called "self-closing" tags which means they do not have the typical structure of an opening and closing tag. Instead, all of the information for the image is specified using attributes inside the tag, and then closed with the ending />. There are many possible attributes to use, but the two that are required are src and alt. 

<img src="url" alt="some_text"/>

src stands for source, the place where you are pulling the image from. The alt tag is short for alternative text, which is displayed if something goes wrong with the image.

If you want to scale your image, then you can also add additional height width attributes, which specify in pixels how high/wide an image is. 

<img src="../alien.jpg" alt="alien" width="150" height="150" />
Troubleshooting...

Getting a 404 (Not Found) Error? Then you haven't saved your source image in the right place, or you do not have the right path for the image! The best place to save your image is in the web folder, the same place as your problem1.html webpage. In your webpage, you can use ./imagename.extension to reference the image. If you are using an online image, be sure the link begins with "http://www."
