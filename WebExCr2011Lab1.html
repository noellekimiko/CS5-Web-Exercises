Web Extra Credit Lab 1 (HTML)

Running a Simple CGI Web Server

To view your webpage use the command-line to run a CGI server. This server can be used to host your webpage locally.

First you will need to create a directory (folder) on your desktop called web, and create another directory inside Web called cgi-bin. cgi-bin is where you will need to save your webpage. To test the server you can copy the following test code into a new python file named hello.py and save it in cgi-bin. 

#!/usr/local/bin/python

print 'Content-Type: text/html'
print
print '<html>'
print '<head><title>Hello from Python</title></head>'
print '<body>'
print '<h2>Hello from Python</h2>'
print '</body></html>'

In case you are not familiar with command-prompt... http://www.cs.princeton.edu/courses/archive/spr05/cos126/cmd-prompt.html

Then open up the command-line prompt (Terminal on a mac). Navigate to your web folder by typing cd Desktop, then typing cd web. After that, paste in the following to open up python. 

python -m CGIHTTPServer
You should then see the following message. 
C:\classes\python>python -m CGIHTTPServer
Serving HTTP on 0.0.0.0 port 8000 ...

The server is now running your page! To view it, open http://localhost:8000/cgi-bin/hello.py in your browser.


Troubleshooting...:


If nothing at all appears...   look back at the terminal window where you started the web server -- there, you may see an error like OSError: [Errno 13] Permission denied. If you do, it's because the hello.py file does not have the permissions necessary to run as an executable file. To fix this, run


chmod 755 hello.py
from within the cgi-bin directory.


Getting an Error Message when starting up your server? If you run into problems hosting your website, the problem might be in the first line of hello.py. Check where your version of Python is saved, and change the file path as necessary. If the command gives you a message like: 

'python' is not recognized as an internal or external command,
operable program or batch file.
or: 
Bad command or filename

...Then you probably need to set your computer to know where the Python interpreter is. Read this PythonPath page to learn how.



Otherwise, the following type of messages should appear in your command line window if things are going well: 

localhost - - [29/Jan/2008 20:13:54] "GET /hello.py HTTP/1.1" 404 -
localhost - - [29/Jan/2008 20:14:17] "GET /cgi-bin/hello.py HTTP/1.1" 200 -
localhost - - [29/Jan/2008 20:14:17] command: C:\Python25\python.exe -u C:\class
es\python\cgi-bin\hello.py ""
localhost - - [29/Jan/2008 20:14:17] CGI script exited OK

If at any time you need to exit the server, type Control-C.

Body and Headers in HTML

HTML uses tags to describe the information on a webpage. Note that and tags are required for every html page. Tags with a forward slash are used to close open tags- if you forget a closing tag, your page will not turn out the way you want it!

Header tags, such as h1 , h2 , and paragraph tags p , distinguish text on the page.

You can copy and paste this code into your problem1.html file to see how these tags work. 

 
<html>
<body>

<h1>My First Heading</h1>

<p>My first paragraph.</p>

</body>
</html>

There are also tags for basic text formatting: 

<p><b>This text is bold</b></p>
<p><strong>This text is strong</strong></p>
<p><big>This text is big</big></p>
<p><i>This text is italic</i></p>
<p><em>This text is emphasized</em></p>
<p><code>This is computer output</code></p>
<p>This is<sub> subscript</sub> and <sup>superscript</sup></p>

Feel free to try these out in your file!

Lists and Tables

There are two types of lists in HTML, ordered lists and unordered lists. Ordered lists number the items, while unordered lists use bullet points.

Unordered lists start with a ul tag, and ordered lists start with a ol tag.

Individual items in the list are denoted by the li tag, which stands for 'list item'.

Here is an example of an unordered list: 

<ul>
<li>aliens</li>
<li>spam</li>
</ul>

Here is an example of an ordered list: 

<ol>
<li>aliens</li>
<li>spam</li>
</ol>

Tables are denoted by the table tag. To add rows to your table, add in a tr table row tag, and add columns by entering table data by adding td tags within your row tags. Again, feel free to play around with your own table. 

<table>
<table border="1">
<tr>
<td>row 1, cell 1</td>
<td>row 1, cell 2</td>
</tr>
<tr>
<td>row 2, cell 1</td>
<td>row 2, cell 2</td>
</tr>
</table>

Links

The html code for link is fairly straightforward: 

<a href="url">Link text</a>

The href specifies the destination of the link, and the a tag stands for 'anchor'.

To change the code above, replace the url and Link text with your desired values.

Images

The html tag for images is img. Since image tags have no closing tag (you put the descriptors in the tag itself) it is called an empty tag. 

<img src="url" alt="some_text"/>

src stands for source, the place where you are pulling the image from. The alt tag is short for alternative text, which is displayed if something goes wrong with the image.

If you want to scale your image, then you can also add additional height width attributes, which specify in pixels how high/wide an image is. 

<img src="../alien.jpg" alt="alien" width="150" height="150" />

Getting a 404 (Not Found) Error? Then you haven't saved your source image in the right place, or you do not have the right path for the image! The best place to save your image is in the Web folder, adjacent to the cgi-bin folder. In your html, you can then call the image with ../imagename.extension. If you are calling your image from a website, be sure the link begins with "http://www."
